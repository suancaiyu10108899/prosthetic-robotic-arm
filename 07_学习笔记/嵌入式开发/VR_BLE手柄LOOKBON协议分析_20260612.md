# VR BLE 手柄 "LOOKBON" 协议分析

> **日期**：2026-06-12
> **主题**：分析第二个手柄（VR BLE 遥控手柄，广播名 "LOOKBON"）的 GATT 协议与数据格式
> **来源**：商家提供的 Gitee 仓库 (`vr-ble` + `vr-ble-arduino` + `vr-ble-python-pc`)

---

## 一、手柄规格速览

| 参数 | 值 |
|------|-----|
| 蓝牙版本 | BLE (4.0+) |
| 角色 | **Peripheral（从机）** |
| 广播名 | `LOOKBON` |
| 输入 | 7按键 + 8方向摇杆 |
| 数据格式 | **2 字节十六进制字符串**（与 CodexPad 的 8 字节不同！） |
| 按键数 | 7个：@(功能) / A / B / C / D / R(侧下) / L(侧上) |
| 摇杆 | 8方向数字输出（无模拟量！无X/Y轴值） |
| 连接方式 | MAC 地址直连 |

## 二、BLE 通信协议

### 2.1 GATT UUID

从 `vr-ble-arduino.ino` 直接提取：

```cpp
static BLEUUID serviceUUID("0000AE30-0000-1000-8000-00805F9B34FB");
static BLEUUID charUUID("0000AE02-0000-1000-8000-00805F9B34FB");
```

| 层 | UUID |
|----|------|
| **Service** | `0000AE30-0000-1000-8000-00805F9B34FB` |
| **Characteristic (Notify)** | `0000AE02-0000-1000-8000-00805F9B34FB` |
| Notify Handle | **7**（Python PC 示例用；另有 131 / 67 备选） |

### 2.2 数据格式 — 2 字节十六进制字符串

每个通知是 **2 字节 ASCII 十六进制字符串**（如 `A1`, `D4`, `B2`），不是二进制数据帧！

```
格式: [事件类型][按键/方向编号]
  第1字节: 事件类型
    'A' = 单击
    'B' = 长按
    'C' = 长按释放
    'D' = 摇杆方向
  第2字节: 按键/方向编号
    '1' = @ 功能键
    '2' = A 键
    '3' = B 键
    '4' = C 键
    '5' = D 键
    '6' = R 键 (侧下)
    '7' = L 键 (侧上)
    '0'~'8' = 摇杆方向 (D系列), 0=中位
```

### 2.3 完整按键映射表

| 数据 | 含义 |
|------|------|
| `A1` | 按键@ 单击 |
| `A2` | 按键A 单击 |
| `A3` | 按键B 单击 |
| `A4` | 按键C 单击 |
| `A5` | 按键D 单击 |
| `A6` | 按键R 单击（侧下） |
| `A7` | 按键L 单击（侧上） |
| `B1`-`B7` | 对应按键 长按 |
| `C1`-`C7` | 对应按键 长按释放 |
| `D0` | 摇杆中位 |
| `D1` | 摇杆 上 |
| `D2` | 摇杆 下 |
| `D3` | 摇杆 左 |
| `D4` | 摇杆 右 |
| `D5` | 摇杆 左上 |
| `D6` | 摇杆 左下 |
| `D7` | 摇杆 右上 |
| `D8` | 摇杆 右下 |

## 三、与 CodexPad-C10 的关键差异

| 维度 | CodexPad-C10 | VR BLE "LOOKBON" |
|------|:---:|:---:|
| 广播名 | `CodexPad-` 开头 | `LOOKBON` |
| 服务 UUID | `0xFFA0` | `0000AE30-...` |
| 特征 UUID | `0xFFA1` | `0000AE02-...` |
| 数据格式 | **8 字节二进制** | **2 字节 ASCII 字符串** |
| 按键编码 | 17位掩码 (uint32) | 事件类型 + 编号 (A1~D8) |
| 摇杆输出 | **模拟量** (0-255, X/Y轴) | **8方向开关量** (无模拟值) |
| 按键数 | 5（C10），17（全系列） | 7 |
| 连接方式 | MAC 直连 / 按键掩码扫描 | MAC 直连 |
| 广播包 | Manufacturer Data (含按键状态) | 标准 BLE 广播 |
| 手柄复杂度 | 低（简单十字键+单摇杆） | 低（按键+8向摇杆，无模拟） |
