## 一、ResearchKit SDK 拆包结果

### Wheel 结构
```
researchkit_py_sdk-1.0.0-cp310-cp310-win_amd64.whl
└── ResearchKit_SDK/
    ├── __init__.py          ← 导出了 SensorType, SensorConfig, SensorData 等
    ├── client.cp310-win_amd64.pyd          ← 主客户端（编译为二进制）
    ├── models.cp310-win_amd64.pyd          ← SensorType 枚举定义（二进制，无法读取）
    ├── ble_manager.cp310-win_amd64.pyd     ← BLE 底层管理
    ├── data_processor.cp310-win_amd64.pyd  ← 数据处理
    ├── rsk_protocol.cp310-win_amd64.pyd    ← 通信协议
    ├── tlv_parse.cp310-win_amd64.pyd       ← TLV 数据解析
    └── exceptions.cp310-win_amd64.pyd      ← 异常定义
```

### 关键发现
- **SensorType 被封在 `.pyd` 二进制里**，无法直接读取枚举值
- SDK 作者：Giland.YIN，版本 1.0.0
- 仅兼容 **Python 3.10 64-bit Windows**（cp310-win_amd64）
- 前人代码只用了 `SensorType.EMG` 和 `SensorType.IMU` 两种类型
- 是否支持力触觉手套的 `FORCE` / `GLOVE` 类型未知，**需要硬件连接后实际运行 SDK 才能确认**

## 二、NOKOV 动作捕捉体系

### 软件栈
| 组件 | 版本 | 路径 |
|------|------|------|
| XINGYING 动捕软件 | 3.4.0.4088 | `NOKOV/NOKOV/XINGYING 3.4.0.4088【安装包】/XINGYING 3.4.0.4088.exe` |

| Python SDK Wheel | nokovpy-3.0.1 | `dist/nokovpy-3.0.1-py3-none-any.whl`（纯 Python，不挑 Python 版本！） |
| Windows C++ SDK | 2.4.0.5156 | 含 DLL + 头文件 + CMake demo |
| C# SDK | 2.4.0.5124 | |
| Linux SDK | 2.4.0.5156 | |

### SDK 使用方式（来自官方示例）
```python
from nokov.nokovsdk import *

client = PySDKClient()
client.PySetDataCallback(py_data_func, None)
client.Initialize(b"10.1.1.198")  # XINGYING 服务器 IP
# 回调中拿 pFrameOfMocapData → markers 3D 坐标 (mm)
```

### 数据能力
- **Marker Set**：追踪每个标记点的 3D 坐标（mm），支持多组 markerset
- **Rigid Bodies**：刚体 6DoF（位置 + 四元数姿态）
- **力板数据**：有接口但前人未用
- **Unidentified Markers**：未分类的标记点
- 官方示例有手指关节角度计算 demo

### vs 前人的实现
- 前人 `NokovRecorder` 类和官方 `Nokov_SDK_Client.py` 高度相似
- 前人加了 `perf_counter_ns()` 时间标签 + EMG 对齐缓冲
- 官方示例更详细地打印了所有数据类型（MarkerSet + RigidBodies + Unidentified）

## 七、已知限制总结

| 限制 | 影响 | 缓解措施 |
|------|------|------|
| ResearchKit 仅 Py3.10 | 当前 Python 3.11 无法安装 | 需装 Python 3.10 并创建 venv |
| NOKOV SDK 需 XINGYING 服务器 | 无动捕硬件无法测试 | 已有示例代码可参考 API 用法 |
| 手套信息缺失 | 无法规划 Linux 集成方案 | 向学长学姐确认型号 |
| SensorType 在 .pyd 中不可读 | 无法提前知道是否支持手套 | 硬件到后用 `dir(SensorType)` 确认 |
| .bak 文件无手套代码 | 前人可能从未集成手套 | 手套集成需从零开始 |
| D435i 未接入前人的同步框架 | 需要新开一个线程 | 架构清晰，加一个 `run_d435i()` 线程即可 |
