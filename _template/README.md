# 项目模板 v1.0

> 🏗️ 新开实验室项目时，复制此目录即可。

## 用法

```bash
# 在 LabDocs 根目录下执行
cp -r _template 新项目名

# 然后做三件事：
# 1. 填 CLAUDE.md（当前阶段 + 设备列表 + 下一步）
# 2. 填 README.md（项目概述）
# 3. 更新根目录 CLAUDE.md 的项目速览表（加一行）
```

## 目录说明

| 目录 | 用途 | 何时创建 |
|------|------|------|
| `CLAUDE.md` | 🤖 AI 入口，每次协作开始先读 | 项目启动时必填 |
| `README.md` | 人类入口，一句话概述 | 项目启动时必填 |
| `任务看板.md` | TODO/DONE/📦 三栏 | 项目启动时创建 |
| `问题追踪.md` | 🔴未解决/🟡进行中/✅已解决 | 遇到第一个问题时创建 |
| `日总结/` | 每天推进一步一个文件 | 按需写 |
| `架构设计/` | 系统拓扑、数据流、协议设计 | 设计阶段 |
| `设备打通记录/` | 每个硬件/传感器的打通日志 | 接入新设备时 |
| `原始资料/` | SDK/手册/发票/驱动/压缩包 | 拿到资料时立即可存 |
| `采购与物料/` | 3D打印/元器件采购记录 | 花钱时 |
| `实验记录/` | 只放 INDEX.md（元数据索引），不存大文件 | 开始采集数据时 |
| `子系统/` | 🆕 多子方向项目的子系统目录（含 README.md + 任务看板.md） | 某子方向 ≥5 个独立任务时 |

## 反模式（别这么做）

- ❌ 项目完结后资料散落到公共区 → 全部跟项目归档到 `_archive/`
- ❌ 实验原始数据（CSV/npy/mp4）放入 Git → 只放 INDEX.md 索引
- ❌ CLAUDE.md 写了就忘 → 每次 AI 协作结束必须更新
- ❌ 日总结写流水账 → 只记"做了什么决定 + 碰到了什么问题"
- ❌ 设备打通记录格式不统一 → 参考已有项目的打通记录格式
