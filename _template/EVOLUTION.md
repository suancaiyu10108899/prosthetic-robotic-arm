# 项目模板演化记录

> 每次从实际项目中提炼改进后，记录到这里。
> 目标：让下一个项目的启动成本越来越低。

---

## v1.0 — 初始版本 (2026-07-15)

**来源**：从假肢机械臂和四设备数据采集的目录结构反向提炼

**核心设计**：
- 每个项目一个独立子目录，含 `CLAUDE.md` + `README.md` 双入口
- 目录分 7 个标准区：日总结 / 架构设计 / 设备打通记录 / 原始资料 / 采购物料 / 实验记录 / 问题追踪
- 原始资料跟项目走（不拆分到公共区）
- 实验记录只放索引，不放大数据
- 模板本身有 `EVOLUTION.md` 跟踪演化，每次提炼都要记录"来源 + 为什么改"

**借鉴**：
- `miniciv` 的 CLAUDE.md 分层记忆（全局 → 项目）
- `dev-hub` 的 06-template-evolution.md（小步提炼、每次只改一个方面）
- `dev-hub` 的 04-decision-log.md（决策记录格式）

**已知不足**（待下个项目验证）：
- [ ] 设备打通记录格式是否太死？实际使用频率如何？
- [ ] 是否需要一个"对外汇报"目录放 PPT/海报？
- [ ] 多项目并行时，日总结要不要加项目标签前缀？
- [ ] 是否需要项目间的"阶段回顾"统一模板？
