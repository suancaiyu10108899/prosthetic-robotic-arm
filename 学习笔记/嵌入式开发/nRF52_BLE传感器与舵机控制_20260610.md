# nRF52 BLE 传感器数据采集与舵机控制

> **日期**：2026-06-10
> **主题**：分析 `sketch_feb3a.ino` 固件代码
> **来源**：项目代码（基于 Bend Labs 示例修改）

---

## 一、学习目标

理解 `sketch_feb3a.ino` 如何通过蓝牙接收弯曲传感器角度数据，处理后驱动舵机。

---

## 二、系统角色

| 角色 | 硬件 | 功能 |
|------|------|------|
| **外设 (Peripheral)** | Bend Labs 弯曲传感器 | 广播角度测量数据 (BLE GATT) |
| **中央 (Central)** | nRF52840 开发板 | 扫描→连接→接收通知→控制舵机 |

nRF52840 作为 **BLE Central**（主机），主动扫描并连接传感器（从机）。

---

## 三、BLE 通信协议

### 服务/特征 UUID
```cpp
BLEClientService        angms(0x1820);  // Angle Measurement Service
BLEClientCharacteristic angmc(0x2a70);  // Angle Measurement Characteristic
```
- `0x1820`：标准 BLE 角度测量服务
- `0x2a70`：标准 BLE 角度测量特征

### 数据包格式（8字节）
```cpp
void ams_notify_callback(... uint8_t* data, uint16_t len) {
    if(len == 8) {
        memcpy(&sample[0], &data[0], sizeof(float));  // X轴 4字节 float
        memcpy(&sample[1], &data[4], sizeof(float));  // Y轴 4字节 float
    }
}
```
每次通知发送 8 字节：`[X角度(float) | Y角度(float)]`

---

## 四、连接流程

```
setup() → 扫描外设 → 发现 UUID=0x1820 → 连接
  ↓
connect_callback() → 发现服务 → 发现特征 → 读型号
  ↓
"ADS_ONE_AXIS" 或 "ADS_TWO_AXIS" → 判断传感器类型
  ↓
enableNotify() → 接收数据 → ams_notify_callback
```

## 五、角度到舵机的映射

### 核心公式
```cpp
ss  = int(sample[0]);        // 传感器原始角度 (浮点→整数)
ss2 = (int)((ss + 250) / 250 * 90);  // 映射到舵机范围
myservo.write(ss2);           // 写入 0-180°
```

### 映射表
| 传感器角度 | 计算过程 | 舵机角度 |
|:---------:|:--------:|:-------:|
| -250° | (-250+250)/250×90 = 0 | 0° |
| 0° | (0+250)/250×90 = 90 | 90° (中位) |
| +250° | (250+250)/250×90 = 180 | 180° |

> 📌 `+250` 偏移把传感器范围从 `[-250,250]` 搬到 `[0,500]`，再按比例缩放到 `[0,180]`

### 舵机连接
```cpp
myservo.attach(A2);  // 舵机信号线接 A2 引脚
```

## 六、标定协议

通过串口（115200 波特率）发送单字符命令：

### 单轴传感器
| 字符 | 含义 | 传感器位置 |
|:---:|------|-----------|
| `0` | 第一步标定 | 0° |
| `9` | 第二步标定 | 90° |
| `b` | 拉伸标定 1 | 0mm |
| `e` | 拉伸标定 2 | 30mm |
| `c` | 恢复出厂 | - |

### 双轴传感器
| 字符 | 含义 | X轴 | Y轴 |
|:---:|------|:---:|:---:|
| `0` | 第一步 | 0° | 0° |
| `f` | 第二步 | 90° | 0° |
| `p` | 第三步 | 0° | 90° |
| `c` | 恢复出厂 | - | - |

> ⚠️ 标定顺序很重要！必须先 0° 后 90°；单轴拉伸标定必须 0mm 先于 30mm。

---

## 七、要点总结

1. nRF52840 是 BLE **主机**（不是从机），主动连传感器
2. 通信协议完全基于标准 BLE GATT，不需要自定协议
3. 8字节数据 = 两个 float（双轴角度）
4. 角度映射是**纯线性**，公式简单可调
5. 标定必须通过**串口键盘**触发，系统启动后不会自动标定
