## 二、TouchGlove SDK 深入

### 基本信息
| 项目 | 值 |
|------|-----|
| SDK | TouchGlove SDK（Python + Rust） |
| 接口 | 串口 `/dev/ttyACM0` |
| 数据类型 | **dense 位移场 + 力场**（5 通道 × 32 × 32 × 3 分量） |
| 推理 | 加密 ONNX 模型 `dense_3ch.enc`，需 NVIDIA GPU + CUDA |
| 力值 | fx, fy, fz (牛顿)，位置 x, y (mm) |
| 校准 | `glove.calibrate()` + 去皮(皮重归零) |
| OS | Ubuntu 22+（有 `.so` 和 `cpython-310-x86_64-linux-gnu.so`） |
| Python | 推荐 3.10 |
| ROS2 | `ros2_dense_fields.py` → Float32MultiArray (5,32,32,3) |

### 核心依赖
```
pip install numpy PySide6 matplotlib cryptography onnxruntime-gpu==1.23.2
nvidia-cuda-runtime-cu12  nvidia-cuda-nvrtc-cu12  nvidia-cublas-cu12
nvidia-cufft-cu12  nvidia-curand-cu12  nvidia-cudnn-cu12
```

### 数据流关键代码（demo.py 原始数据）
```python
# 逐帧原始力值（5 通道，f=g(N)，pos=mm）
print(f"CH {frame.channel} | Force: ({fx:.1f}, {fy:.1f}, {fz:.1f}) N | Pos: ({x:.1f}, {y:.1f}) mm")
```

### ~500 MB/min 手套数据量
```
5 channels × 32 × 32 × 3 floats × 4 bytes × 60 fps ≈ 3.7 MB/s ≈ 220 MB/min（dense 场）
+ 另加 disp 场 ≈ 220 MB/min
→ 实时统计约 440 MB/min（以 60 Hz 估计）
```

### 与之前的错误假设对比
| 之前假设 | 实际 |
|---------|------|
| 24×31 触觉矩阵 | **32×32 dense 力场 + 位移场**（5通道） |
| 多种独立数据流 | **单一的 dense 推理输出**（加密 ONNX） |
| 无需 GPU | **需要 NVIDIA GPU + CUDA** |
| 轻量 | **本体系数据量最大的设备** |
