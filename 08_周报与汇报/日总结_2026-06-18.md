## 二、技术突破

### 1. nRF52 UART TX 最终确认

```
引脚: D0 (P0.25)
协议: Serial1 @ 115200 8N1
帧: 0xAA / dir / 0xBB (dir: A→01, B→02, C→03, D→04)
驱动: 高驱动 H0H1 (~5mA, NRF_P0->PIN_CNF[25] |= (3 << 8))
空闲电平: 3.06V (万用表 DC)
示波器: 3 次确认（双引脚交替 + 纯 TX + v2.9 全链路）
```

### 2. NimBLE-Arduino 5 条关键经验（无价）

| # | 经验 | 影响 |
|:--:|------|------|
| 1 | `connect()` 不能写在扫描回调里 — NimBLE 栈锁 | 连接握手超时 |
| 2 | `getCharacteristics(false)` 只取缓存 — 发现阶段不填充 | 0 个 characteristic |
| 3 | 服务发现需要 4 秒延迟 — GATT 表同步慢 | `[SUB] total=0` |
| 4 | `dev->getName()` → `std::string` — 不是 Arduino String | 编译失败 |
| 5 | `canNotify()` 可能返回 false 但 Notify 仍可接收 | 伪阴性 |

### 3. LOOKBON GATT 结构（ESP32 UUID dump 首次完整揭示）

```
0x1800  Generic Access
  └ 0x2a00  Device Name (Read)

0xae30  Custom Service 1
  ├ 0xae01 (Read)
  ├ 0xae02 (Notify)  ← 按键数据
  ├ 0xae03 (Read)
  ├ 0xae04 (Notify)
  ├ 0xae05 (Read)
  └ 0xae10 (Write)

0xae3a  Custom Service 2
  ├ 0xae3b (Read)
  └ 0xae3c (Notify)

0xae00  Custom Service 3
  ├ 0xae01 (Read)
  └ 0xae02 (Notify)
```

## 三、硬件教训

| # | 事故 | 根因 | 教训 |
|:--:|------|------|------|
| 1 | TX(A2) 误焊到 ③号板 5V | 没看引脚定义就焊 | 焊接前万用表蜂鸣档逐脚验连通 |
| 2 | 5V 倒灌 VDD 轨 → GPIO 输入管全部击穿 | 输出端灌入高压, ESD 二极管正向导通 | GPIO 输入端比输出端更脆弱 |
| 3 | **12V 带电焊接灌入 VDD → 板子报废** | **忘记断电** | **焊接第一原则：断电** |
