## 三、解决方案 / 思路

- **硬编码 CCCD Handle 的原因**：LOOKBON 手柄不响应 Adafruit 库的 GATT discover → 从 Python bleak 库逆向拿到 Handle 7/67/131 → 直写 Nordic SoftDevice 底层 API。这不是偷懒，是对不兼容硬件的必要妥协
- **广播名匹配 vs MAC 匹配**：选广播名 `"LOOKBON"` 匹配而非 MAC 硬编码 → 换同型号手柄自动连，换不同型号手柄改一行字符串
- **调试打印保留**：当前阶段保留所有 `Serial.print` 是必要的——学长联调时也需要通过串口确认板子工作正常。产品阶段可全部删除

## 六、学习收获

1. **nRF52840 原生 USB 的 `!Serial` 陷阱**：无 Host 连接时 `!Serial` 永远为 `true`，导致电池供电时板子卡死。3 秒超时是嵌入式常用模式
2. **外部代码评审必须本地核实**：reviewer 引用的 `platformio.ini:35` 行号在实际文件中不存在 → 说明他读了错误版本。逐条本地确认后才能决定是否修改
3. **BLE Central 的 CCCD 订阅流程**：discover→ enableNotify→ 回调 这个标准流程对不兼容的手柄芯片会失败 → 跳过 discover 直写 CCCD Handle 是有效的绕过方案
4. **串口隔离**：`Serial` 走 USB → 电脑，`Serial1` 走引脚 → ③号板。两条物理链路独立，调试日志不会污染③号板数据
5. **项目管理节奏**：固件开发 3 天打通全链路 → 文档同步贯穿始终 → 发票归档不留空白 → 周报/日总结/看板/AI_MEMORY 四层管理
