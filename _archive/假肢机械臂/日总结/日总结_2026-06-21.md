## 二、代码改动 — v2.10 长按区分

### 需求
学长提出长按应发送不同方向码（0x05-0x08），让③号板 STM32 识别长按后持续顶到限位。

### 改动

| 文件 | 改动 |
|------|------|
| `handle_parser.cpp` | `event == 0xA \|\| event == 0xB` 拆为两个分支，长按偏移 +8 位（bit12-15） |
| `main.cpp` sendUartFrame() | 新增长按映射 0x05-0x08，版本号 → v2.10 |
| `README.md` | 方向码表扩展到 8 条 |

### 最终方向码表

| 按键 | 单击 | 长按 |
|:--:|:--:|:--:|
| A | `0x01` | `0x05` |
| B | `0x02` | `0x06` |
| C | `0x03` | `0x07` |
| D | `0x04` | `0x08` |

## 三、远程协助学长征

### 环境搭建

- 学长安了 Anaconda 但未加入 PATH → 用 `A:\python\anaconda\python -m pip install platformio` 安装
- 首次编译下载工具链 + nRF52 框架 + Bluefruit 库约 10 分钟
- 用 `github.com/suancaiyu10108899/arm-ble-firmware` 公开仓库

### 烧录过程

- 学长用 ZIP 下载 → 解压到 `A:\arm-ble-firmware`
- 首次烧录的是旧 v2.9（下载时仓库还未更新）
- `git pull` 不可用（未装 Git）→ 重新下载 ZIP 覆盖
- 覆盖后缓存问题导致 BLE 连接失败 → `platformio run --target clean` + 重新烧录解决
- 最终 v2.10 烧录成功，BLE 连接正常，D4 蓝灯亮

### 经验教训

- 给非技术背景的人，README 步骤要写到"傻瓜级"
- "5 分钟上手"里的 `git clone` 假设对方有 Git——实际很多人没有
- PlatformIO 首次下载工具链很慢，要提前打预防针
- 覆盖代码后最好做 clean build
