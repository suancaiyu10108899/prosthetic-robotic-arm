## 5. 关键技术发现

### Bluefruit.begin 参数顺序坑
`Bluefruit.begin(peripheralCount, centralCount)` — 第一个参数是 Peripheral，第二个才是 Central。

### 纯 Central 模式扫描器 bug
`begin(0,1)` 纯 Central 下 onScan 只触发一次后永久休眠 → `begin(1,1)` + loop 重启绕过。

### GATT 发现不兼容
Adafruit 库 `BLEClientService::discover()` 与 LOOKBON 不兼容 → 参照 Python 代码跳过 discover，直写 CCCD。

### SoftDevice 中断上下文阻塞
`onConnect` 回调在 SoftDevice 中断上下文中执行，`delay(2000)` 阻塞事件处理 → CCCD 写入移到 loop 中。

### LOOKBON 协议：单字节编码
高 nibble = 事件类型 (0xA=单击/0xB=长按/0xC=释放/0xD=方向)，低 nibble = 按键/方向编号。

### `while (!Serial)` 死等电池供电 bug (v2.7 关键教训)
nRF52840 原生 USB CDC 在无 Host 时 `!Serial` 永远为 true，导致电池供电时板子卡在 setup 开头不启动。改为 3 秒超时。**发现过程**：先手动烧录确认 MicroUSB 正常 → 不打开串口监视器观察到蓝灯不亮 → 读代码定位到 `while (!Serial)` → 改为超时。这个 bug 是靠自己排查发现的，AI 在写代码和初步分析时都没注意到。
